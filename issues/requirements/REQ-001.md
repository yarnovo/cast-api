## 描述

老板要 agent harness 支持 chat session 持久化 (跨 FC 实例 · agent 多轮对话不丢上下文)。
FC 无状态 · jsonl/sqlite 撑不住跨 instance · 走 RDS。

新建 `chat_messages` 表 (跟 `agent_memories` 分开 · chat 是短期 turn · memory 是长期沉淀)。

## acceptance

1. `chat_messages` 表落库 · 字段完整 (id/session_id/agent_id/user_id/role/content/content_json/tool_call_id/tool_name/created_at/metadata_json)
2. 复合索引 `(session_id, created_at)` + 单列索引 `(agent_id, created_at)` 真存在
3. `uv run alembic upgrade head` 本地跑过 · `downgrade -1` 双向跑过
4. `POST /api/chat_messages` `GET /api/chat_messages?session_id=...` `DELETE /api/chat_messages?session_id=...` 三个 endpoint 真 200
5. pytest 覆盖 3 endpoint + alembic upgrade/downgrade · `uv run pytest -v` 全绿
6. README Schema 段加 chat_messages · CONTRACTS.md 声明 akong-agent-harness 下游

- 不"统一" chat_messages 跟 agent_memories (两层抽象)
- 不实现 RdsSession 客户端 (那是 akong-agent-harness 的事 · 本仓只提供 endpoint)
